# 64 点基-4 SDF 流水线 FFT 设计规格书

## 1. 任务综述

设计并实现一个基于 **SDF (Single-path Delay Feedback)** 架构的 **64 点基-4** 快速傅里叶变换（FFT）逻辑模块。该设计需支持连续数据流输入，并采用流水线结构以最大化吞吐量。

---

## 2. 核心设计参数

| 参数项 | 规格要求 | 备注 |
| --- | --- | --- |
| **点数 (N)** | 64 点 | $\log_4(64) = 3$ 级阶段 |
| **基数 (Radix)** | Radix-4 | 每一级处理 4 个采样点 |
| **输入顺序** | 自然顺序 (Natural Order) | 连续串行输入 |
| **输出顺序** | 基-4 位反转顺序 (Digit-reversed) | 流水线架构典型特征 |
| **数据格式** | 定点数 (Fixed-point) | 16-bit 实部 + 16-bit 虚部 |
| **数值精度** | Q1.15 格式 | 范围 [-1, 1) |

---

## 3. 详细架构要求

### 3.1 级联结构 (Pipelined Stages)

设计必须分为三个物理阶段（Stage 0, Stage 1, Stage 2）：

1. **Stage 0**: 处理 $N=64$。反馈移位寄存器跨度为 $16, 32, 48$。需进行旋转因子乘法。
2. **Stage 1**: 处理 $N=16$。反馈移位寄存器跨度为 $4, 8, 12$。需进行旋转因子乘法。
3. **Stage 2**: 处理 $N=4$。反馈移位寄存器跨度为 $1, 2, 3$。**无需**进行旋转因子乘法。

### 3.2 关键子模块规格

* **基-4 蝴蝶单元 (BF4)**：
* 实现以下变换：

$$Z_0 = (x_0 + x_2) + (x_1 + x_3)$$


$$Z_1 = (x_0 - x_2) - j(x_1 - x_3)$$


$$Z_2 = (x_0 + x_2) - (x_1 + x_3)$$


$$Z_3 = (x_0 - x_2) + j(x_1 - x_3)$$


* 注意：与 $-j$ 的乘法应实现为实虚部互换及符号取反，不得消耗乘法器资源。


* **旋转因子 (Twiddle Factors)**：
* 使用 ROM 存储 $\cos$ 和 $\sin$ 值。
* 每一级需根据当前的计数器值生成正确的 ROM 地址。


* **复数乘法器**：
* 支持 $ (a+bi) \times (c+di) $。
* 由于是高频设计，乘法器内部需包含至少 1 级流水线寄存器。



### 3.3 位宽控制与溢出

* **位宽扩展**：每一级蝶形运算允许内部中间值位宽增加 2 bits。
* **舍入处理**：在每一级输出至下一级前，必须进行收缩处理（通常右移 2 bits）。
* **策略**：推荐使用四舍五入（Rounding）而非直接截断（Truncation），以保证最终的信噪比（SQNR）。

## 4. 接口定义 (Verilog Interface)

```verilog
module fft64_radix4_sdf (
    input  wire                clk,          // 系统时钟
    input  wire                rst_n,        // 异步复位（低有效）
    
    // 输入接口
    input  wire                in_valid,     // 输入数据有效标志
    input  wire signed [15:0]  in_data_re,   // 输入实部 (Q1.15)
    input  wire signed [15:0]  in_data_im,   // 输入虚部 (Q1.15)
    
    // 输出接口
    output wire                out_valid,    // 输出数据有效标志
    output wire signed [15:0]  out_data_re,  // 输出实部 (Q1.15)
    output wire signed [15:0]  out_data_im,  // 输出虚部 (Q1.15)
    output wire [5:0]          out_index     // 输出数据索引 (0-63)
);

```

## 5. 验收方法 (Acceptance Criteria)

### 5.1 功能与精度 (Precision)

* **测试向量**：输入随机复数序列。
* **对比标准**：将 RTL 输出与 Python `scipy.fft` 或 `numpy.fft` 计算结果（转换为定点后）进行对比。
* **容差**：所有样点的平均量化误差需 $< 1\%$，或 SQNR（信号量化噪声比）$> 60\text{dB}$。

### 5.2 时序与吞吐量 (Throughput)

* **吞吐量**：在 `in_valid` 持续拉高的情况下，模块必须每个时钟周期处理一个复数样点，不得有停顿（Stall）。
* **延迟 (Latency)**：从第一个样点输入到第一个有效结果输出，延迟应保持恒定（由流水线深度决定）。

### 5.3 资源优化检查

* **旋转因子优化**：Stage 2 是否避免了无效的乘法器调用？
* **复数乘法优化**：是否使用了高效的复数乘法结构（如 3 乘法器方案）？
