## 一、 系统指令与任务背景
你现在是一位资深的 IC 设计工程师，精通 Verilog 和密码学硬件加速。你的任务是编写一个标准、可综合、面积优化的 SHA-256 硬件核心代码。

## 二、 顶层接口定义 (Top-Level Interface)
你的顶层模块名必须为 `sha256_core`，接口定义如下：
```verilog
module sha256_core (
    input  wire           clk,          // 系统时钟
    input  wire           reset_n,      // 异步低电平复位
    input  wire           init,         // 高电平脉冲：初始化哈希状态（加载初始向量 IV）
    input  wire           next,         // 高电平脉冲：指示 block 输入有效，开始处理新的 512-bit 数据块
    input  wire [511:0]   block,        // 512-bit 消息块输入
    output reg            ready,        // 高电平：核心处于空闲状态，可以接收 init 或 next
    output reg            digest_valid, // 高电平脉冲：指示 digest 输出的 256-bit 结果当前有效
    output reg  [255:0]   digest        // 256-bit 哈希结果输出
);
```

## 四、 核心组合逻辑与函数 (Functions)
在模块内部，请优先使用 `function` 或 `wire` 组合逻辑来实现以下按位操作：

**1. 循环右移 (ROTR)**
SHA-256 的核心是 32 位无符号整数的循环右移。请确保移位操作是针对 32 位宽的，即：
`ROTR(x, n) = (x >> n) | (x << (32 - n))` （在 Verilog 中可通过拼接实现：`{x[n-1:0], x[31:n]}`）。

**2. 逻辑运算函数**
请实现以下组合逻辑：
* `Ch(x,y,z) = (x & y) ^ (~x & z)`
* `Maj(x,y,z) = (x & y) ^ (x & z) ^ (y & z)`
* `Sigma0(x) = ROTR(x, 2) ^ ROTR(x, 13) ^ ROTR(x, 22)`
* `Sigma1(x) = ROTR(x, 6) ^ ROTR(x, 11) ^ ROTR(x, 25)`
* `sigma0(x) = ROTR(x, 7) ^ ROTR(x, 18) ^ (x >> 3)`
* `sigma1(x) = ROTR(x, 17) ^ ROTR(x, 19) ^ (x >> 10)`

## 五、 硬件架构与数据通路 (Datapath)

**1. 消息调度模块 (W 数组生成)**
* **设计：** 使用一个包含 16 个 32位寄存器的滑动窗口（Shift Register 数组），命名为 `W[0:15]`。
* **大端序字长映射：** 当 `next` 为高时，将输入的 512 位 `block` 载入到 `W` 数组的前 16 个字中。注意 SHA-256 是大端序：
    `W[0] = block[511:480]; W[1] = block[479:448]; ... W[15] = block[31:0];`
* **W 数组更新：** 在第 $t$ 轮运算中，当前轮所需的 $W_t$ 逻辑如下：
    * 如果 $t < 16$，当前参与计算的 $W_t$ 就是移位寄存器的最前端值 `W[0]`。
    * 如果 $t \ge 16$，计算新值：$W_{new} = \sigma_1(W[14]) + W[9] + \sigma_0(W[1]) + W[0]$。
    每个时钟周期结束时，W 数组向左平移（`W[i] <= W[i+1]`），并将 $W_{new}$（或输入的新数据）推入 `W[15]`。

**2. 压缩函数模块**
* 定义 8 个工作寄存器：`a, b, c, d, e, f, g, h`。
* 定义组合逻辑中间变量：`T1` 和 `T2`。
    `T1 = h + Sigma1(e) + Ch(e, f, g) + K_t + W_t`
    `T2 = Sigma0(a) + Maj(a, b, c)`
* 寄存器更新（时序逻辑）：在运算状态下，每个时钟上升沿更新：
    `h <= g; g <= f; f <= e; e <= d + T1; d <= c; c <= b; b <= a; a <= T1 + T2;`

## 六、 有限状态机设计 (FSM)
请设计一个简单的 FSM 来控制整个流程，状态定义如下：

1.  **IDLE (00)**：
    * 如果 `init == 1`，将内部状态寄存器 `H0~H7` 重置为 `H0_INIT~H7_INIT`。保持 `ready = 1`。
    * 如果 `next == 1`，将 `block` 数据载入 `W` 数组，将 `H0~H7` 赋值给 `a~h` 寄存器，计数器 `t` 清零，跳转到 **COMPUTE** 状态。
2.  **COMPUTE (01)**：
    * 保持 `ready = 0`。每过一个时钟周期，计数器 `t` 加 1。
    * 执行一次工作寄存器（`a~h`）的压缩与更新，同时更新 `W` 数组。
    * 当 `t == 63` 时，说明 64 轮压缩完毕，跳转到 **UPDATE_H** 状态。
3.  **UPDATE_H (10)**：
    * 将工作寄存器的结果累加到中间哈希状态寄存器中：`H0 <= H0 + a; ... H7 <= H7 + h;`
    * 单周期状态，直接跳转到 **DONE** 状态。
4.  **DONE (11)**：
    * 将 `H0~H7` 拼接赋值给 `digest` 输出。拉高 `digest_valid` 一个周期。
    * 跳转回 **IDLE** 状态。

## 七、 核心伪代码流程 (Pseudocode)
硬件的执行流程应严格对齐以下处理单块 512-bit 的伪代码逻辑：

```text
// 1. 初始化阶段 (init)
H0~H7 = H_INIT

// 2. 准备阶段 (next)
W[0..15] = block 的按 32-bit 大端序拆分
a, b, c, d, e, f, g, h = H0, H1, H2, H3, H4, H5, H6, H7

// 3. 64 轮压缩 (COMPUTE 状态)
For t from 0 to 63:
    if t < 16:
        Wt = W[t]
    else:
        Wt = sigma1(W[t-2]) + W[t-7] + sigma0(W[t-15]) + W[t-16]
        
    T1 = h + Sigma1(e) + Ch(e, f, g) + K[t] + Wt
    T2 = Sigma0(a) + Maj(a, b, c)
    
    h = g; g = f; f = e; e = d + T1;
    d = c; c = b; b = a; a = T1 + T2;

// 4. 更新哈希 (UPDATE_H 状态)
H0 = H0 + a; H1 = H1 + b; ... H7 = H7 + h;

// 5. 输出 (DONE 状态)
digest = {H0, H1, H2, H3, H4, H5, H6, H7}
```

## 八、 严格的设计防错与综合规范 (Guardrails)
1.  **时序与组合逻辑分离：** 不要在组合逻辑（`assign` / `function`）内部使用阻塞赋值更新状态。所有的 `always @(posedge clk)` 必须使用非阻塞赋值 `<=`，组合逻辑必须无锁存器（Latch-free）。
2.  **加法溢出：** SHA-256 的所有加法均为**模 $2^{32}$ 加法**。在 Verilog 中，对 32 位宽的数据直接使用 `+` 相加即可天然实现模 $2^{32}$ 加法，无需多余操作。
3.  **复位逻辑：** `reset_n` 为低电平时，必须能将 FSM 强制拉回 IDLE 状态，并将计数器、`ready` 等关键控制信号重置。
