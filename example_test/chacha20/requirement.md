#### 1. 任务背景与目标
* **目标**：实现 ChaCha20 加密与解密算法（基于 RFC 8439 标准）。
* **特性**：流密码。加密和解密过程完全一致（都是将明文/密文与生成的密钥流进行按位异或）。
* **核心机制**：通过一个 256 位（32 字节）的密钥、一个 96 位（12 字节）的随机数（Nonce）和一个 32 位（4 字节）的计数器，生成任意长度的伪随机密钥流。

#### 2. 核心数据类型与参数
* **基本运算单位**：32 位无符号整数（`uint32`）。所有的加法运算都必须是在 modulo $2^{32}$ 下进行的。
* **常数（Magic Constants）**：16 个字节的固定字符串 `"expand 32-byte k"`，拆分为 4 个 `uint32` 小端序整数：
    * `0x61707865`, `0x3320646e`, `0x79622d32`, `0x6b206574`
* **密钥（Key）**：32 字节（被转换为 8 个 `uint32` 小端序整数）。
* **计数器（Counter）**：初始值通常为 0 或 1（1 个 `uint32` 小端序整数）。
* **随机数（Nonce）**：12 字节（被转换为 3 个 `uint32` 小端序整数）。
* **块大小（Block Size）**：每次生成 64 字节（512 位）的密钥流。

#### 3. 数据结构：状态矩阵 (State Matrix)

#### 4. 核心算法：四分之一轮函数 (Quarter Round)
这是 ChaCha20 最底层的运算。定义一个函数 `QR(a, b, c, d)`，它接收矩阵中的 4 个索引，并对它们指向的值进行原地（In-place）修改。

运算包含加法（模 $2^{32}$）、按位异或（$\oplus$）和循环左移（$\lll$）。
具体公式如下：
$$a = (a + b) \pmod{2^{32}}$$
$$d = (d \oplus a) \lll 16$$
$$c = (c + d) \pmod{2^{32}}$$
$$b = (b \oplus c) \lll 12$$
$$a = (a + b) \pmod{2^{32}}$$
$$d = (d \oplus a) \lll 8$$
$$c = (c + d) \pmod{2^{32}}$$
$$b = (b \oplus c) \lll 7$$

#### 5. 轮函数与块生成 (Block Generation)
生成一个 64 字节块的完整过程如下：
1.  **初始化**：根据第 3 节的规则填充 16 个元素的初始状态数组。**复制一份这个初始状态**，我们称之为 `working_state`。
2.  **执行 20 轮迭代**：ChaCha20 需要进行 20 轮运算（10 次双轮）。
    * **奇数轮（列运算）**：
        `QR(0, 4, 8, 12); QR(1, 5, 9, 13); QR(2, 6, 10, 14); QR(3, 7, 11, 15);`
    * **偶数轮（对角线运算）**：
        `QR(0, 5, 10, 15); QR(1, 6, 11, 12); QR(2, 7, 8, 13); QR(3, 4, 9, 14);`
3.  **加上初始状态**：20 轮结束后，将 `working_state` 的每一个元素与第 1 步保存的**初始状态**对应元素相加（模 $2^{32}$）。
4.  **序列化**：将得到的 16 个 32 位整数，全部以**小端序（Little-Endian）**转换为 64 个字节。这就是生成的密钥流块。

#### 6. 加密/解密协议流程
1.  接收输入：明文（或密文）字节数组、32 字节 Key、12 字节 Nonce、初始 Counter。
2.  循环处理明文，每次处理 64 字节：
    * 使用当前 Key, Nonce, Counter 生成一个 64 字节的密钥流块。
    * 将明文块与密钥流块进行按位异或（XOR），得到密文块。
    * 将 Counter 加 1。
3.  如果最后一块不足 64 字节，截取密钥流块的前 $N$ 个字节进行异或即可。
